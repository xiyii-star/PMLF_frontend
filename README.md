![阿里云](aliyun.png)


本任务旨在构建一个基于引文知识图谱的端到端智能科研idea系统实现，在给定单一研究主题关键词的条件下，能够自动化完成“检索-阅读-分析-生成”全流程，对海量文献进行深层语义抽取与逻辑关联挖掘，并输出包含交互式图谱、结构化综述报告及科研idea生成的完整科研。

该系统定位为科研工作流中的技术演化洞察与逻辑生成层，其核心价值在于验证：能否跨越浅层的引用统计，深度挖掘文献间“缺陷-克服”、“方法-扩展”的逻辑因果。系统基于“创新并非凭空产生，而是原有技术不断迭代的延续”这一核心理念，通过重构整个 Topic 的演化历程与技术脉络，实现以史为鉴的高可行性科研 Idea 自动生成，从而精准攻克科研链路中“从 0 到 1”的选题生成难题，为最终构建全自动化科研闭环奠定坚实的逻辑基石。

一、实用性：科研与工作的“超级外脑”，真正开箱即用

- 应用价值核心：以技术演化逻辑为驱动的端到端自动化，从检索与阅读到综述与选题生成，显著提升脑力劳动者的产出效率与决策速度。
- 全流程自动化（End-to-End）：EvoNarrator解决了“海量文献读不完、理不清”的痛点。系统自动完成文献检索与去噪、深层因果挖掘与演化重构、结构化综述撰写、启发式Idea生成，用户无需在不同工具间切换，即可完成从数据获取到报告交付的闭环。
- 开箱即用（Web/CLI 双形态部署）：前后端一体化部署，用户可在网页端直接输入研究主题并一键启动分析，实时查看推理进度与结果；亦可通过命令行一行指令生成完整报告。两种形态均输出交互式知识图谱、演化路径与瓶颈分析，以及结构化调研报告与可操作的Idea列表，可直接用于组会汇报、立项论证与论文写作参考。

二、创意性：从“漂亮的呈现”到“有逻辑的创新”

- 逻辑进化引擎（Socket Matching）：超越“谁引用谁”的静态网络，通过 Problem/Method/Limitation/Future_work 四元语义抽取，判断论文之间的因果关系是“克服缺陷”“扩展方法”还是“兑现未来工作”，把引用关系从线条升级为可解释的逻辑链条。
- Idea 生成的全流程管线：从论文检索与深度信息提取到引用关系判断，再到叙事性演化路径识别并结合思维链对可行性进行推导，输出可落地的科研idea。
- 系统在可视化与交互上实现一体化体验：用户在网页端点选主题与参数即可一键生成该领域的论文引用网络与技术演化脉络。交互式引文图谱清晰呈现领域随时间的演化，构建过程可实时回看推理日志。

三、技术深度：严谨架构 + 工程可落地

- 多智能体协作（Navigator → Extractor → Critic → Synthesizer）：引入 Critic 角色进行证据对齐、事实校验与质量打分，结合基于规则与检索增强的约束，显著降低幻觉率。在内部标注集上，信息抽取的 ROUGE/Precision 指标均显著提升。
- 复杂检索策略（8步雪球法）：融合前向/后向引用、共被引挖掘、关键词扩展与主题聚类，并做 arXiv + OpenAlex 跨库映射与去重归一化，保证覆盖度与时序完整性，避免信息茧房与样本偏置。
- 演化识别与证据链：对 Problem/Method/Limitation/Future_work 进行四元语义抽取，重构“克服缺陷/扩展方法/兑现未来工作”的因果链条；将证据片段与引文路径绑定，提升综述与 Idea 的可解释性与可追溯性。
- 前后端工程化落地：后端采用 FastAPI + WebSocket 的流式回传与长任务可观测；Docker 多阶段构建并部署到阿里云 ECS，Nginx 负责 HTTPS 终止与反向代理，完备的 CORS/SSL/TLS 配置消除混合内容风险。前端基于 Next.js 静态导出部署至阿里云 ESA（边缘加速），客户端路由回退策略支持运行时 Task ID；统一 Canvas/WebGL 视觉与交互图谱，全球访问快速稳定。

EvoNarrator 提供开箱即用的前后端一体化体验，用户无需修改代码或额外部署即可直观生成领域的引用网络与技术演化脉络；在教学与科研协作中具有良好的实用性与可分享性，便于在社群中快速传播与复用。
